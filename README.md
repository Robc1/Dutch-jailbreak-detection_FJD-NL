# <center> Dutch-jailbreak-detection (FJD-NL) </center>

## Abstract
Large Language Models (LLMs) are increasingly deployed and used in sensitive contexts. Although alignment training provides some protection against jailbreak attacks, which are designed to bypass safety mechanisms and elicit unsafe or unintended outputs despite the presence of security controls, existing detection methods are often computationally costly. The Free Jailbreak Detection (FJD) method proposes a low-cost alternative by utilizing the first-token confidence scores of the LLM to distinguish jailbreak from benign prompts. The LLM already computes these scores during normal inference, so this detection method requires no additional forward passes. However, FJD was developed and validated exclusively on English datasets, leaving its applicability to other languages unknown. This research answers the following research question: "How effective is confidence-based first-token jailbreak detection when applied to Dutch-language prompts?" It also aims to provide insight into the mechanisms underlying the overall performance of the FJD detection method in this context.

## Dataset
[WildChat-1M](https://huggingface.co/datasets/allenai/WildChat-1M) was used to source 500 benign natural prompts. 500 benign prompts were translated from the 80,000 over-refusal prompts of the [OR-Bench](https://huggingface.co/datasets/bench-llm/or-bench) dataset. The benign dataset used in the [FJD paper](https://aclanthology.org/2025.findings-emnlp.309/#) was also used in this research, but limited to 500 prompts, for a total of 1,500 benign prompts. These prompts originate from [PureDove](https://huggingface.co/datasets/LDJnr/Pure-Dove), [Open-Platypus](https://huggingface.co/datasets/garage-bAInd/Open-Platypus), and [SuperGLUE](https://huggingface.co/datasets/aps/super_glue). The original 500 benign prompts were reused in this study to preserve a shared subset with the FJD paper.

The [WildJailbreak](https://huggingface.co/datasets/allenai/wildjailbreak) dataset contains 2.000 samples of harmful adversarial queries. 1,414 of these samples were used in the jailbreak part of the dataset. The In-Context and Code Injection categories were generated by using [AdvBench](https://huggingface.co/datasets/walledai/AdvBench) prompts as a basis and an LLM to place them into certain prompt templates. For the Obfuscated category, system prompt and preambles for the input prompts from research by [Yuan et al.](https://arxiv.org/abs/2308.06463) were used.

## Setup

Create and activate a conda environment, then install dependencies including FastChat:

```bash
conda create -n fjd python=3.9.19
conda activate fjd
pip install -r requirements.txt
git clone https://github.com/lm-sys/FastChat.git
cd FastChat
pip install -e .
```

## Configuration

Models are defined in `config.py`. Example:

```python
MODELS = {
    "llama3-8b": {
        "name": "C:/Users/[user]/models/llama3-8b",  # local path
        "template": "llama-3",
        "temperature-ft": 1.0,
        "temperature-fjd": 1.0
    },
    "chat-model": {
        "name": "model",
        "template": "vicuna",
        "temperature-ft": 1.0,
        "temperature-fjd": 1.0
    }
}
```

- `name`: local models require the filesystem path to the model
  weights. For API-based models this value is unused.
- `template`: the FastChat conversation template to use. Ignored in
  API mode.
- `temperature-ft` / `temperature-fjd`: see main thesis, Appendix A.2,
  for how these are selected at runtime via `--instruction` and
  `--temperature`.

## Running the scoring step

Set your API key (if using an API-based model) as an environment variable:

```powershell
$env:API_KEY="[INSERT API KEY HERE]"
```

Then run:

```bash
python get_scores.py --model [model-name] --data [path to input prompt files folder] --output [path to output folder] --instruction True --temperature True --use-api
```

Without an API key (local model):

```bash
python get_scores.py --model [model-name] --data [path to input prompt files folder] --output [path to output folder] --instruction True --temperature True
```

## Running detection metrics
**First, leave the conda environment!**

Experiment 1:
For the `--data` argument, one should add the relative path to the parent folder of output results of the model, which in the example below is "chat-model". All of the affirmative prompt folders (e.g. "chat-model\Je bent een goede assistent") are located in the parent folder ("chat-model"). E.g.: python detection.py --model chat-model --data " \data\result\chat-model" --experiment 1

```bash
python detection.py --model [model-name] --data [path to parent output folder] --experiment 1 --exclude-refusals
```

Experiment 2:
For the `--data` argument, one should add the relative path to the specific folders of affirmative prompts on which experiment 2 was conducted, which in the example below is "Je bent een goede assistent". E.g.: python detection.py --model chat-model --data ".\data\result\chat-model\Je bent een goede assistent" --experiment 2 

```bash
python detection.py --model [model-name] --data [path to parent output folder]\[specific prompt] --experiment 2 --exclude-refusals
```

`--exclude-refusals`: when set, excludes refusal prompts from metric computation and outputs heatmaps only. When omitted, all prompts are included and full metric tables are output in addition to heatmaps.
